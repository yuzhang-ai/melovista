# 欣赏模式实施方案

> 实施状态：V1 已于 `v0.9.0` 落地，后续版本完成通用导入、四首内置曲目、环境声、Vercel 主站、《花海》识谱精修与三种播放模式；`v0.15.0` 增加本地文件夹曲库和六音区触控，`v0.16.0` 增加 Web Audio 节拍器与 A–B 循环。当前已完成 MIDI 自动演奏、动态本地队列、节奏训练与用户跟弹。

## 1. 目标

在不影响自由演奏低延迟的前提下，增加一个“欣赏模式”：用户可以从内置纯钢琴曲库选曲并自动演奏，也可以在自动演奏过程中继续使用当前电脑键盘跟弹。

第一版不做流媒体 MP3 播放，而采用 MIDI 音符事件驱动现有钢琴采样。这样自动演奏能复用当前音色、琴键按下效果和场景光尘，也能准确区分“曲目音符”和“用户音符”。

## 2. 推荐体验

### 模式入口

- 顶部“模式”增加二级选择：`自由演奏` / `欣赏模式`。
- 进入欣赏模式后，底部琴键继续可弹，`Space` 和左 `Alt` 继续有效。
- 沉浸模式规则不变；播放器控制通过鼠标操作，不额外开放无关键盘快捷键。

### 播放器

- 曲目：封面色块、曲名、作曲家、时长与授权来源。
- 控制：上一首、播放/暂停、下一首、进度，以及顺序播放、单曲循环、随机播放三种模式。
- 连播：顺序播放到曲库末尾后回到第一首；随机播放排除当前曲目，避免连续重复。
- 速度：`0.75× / 1.0× / 1.25× / 1.5× / 2×`，速度变化不改变音高。
- 混音：自动演奏音量与用户演奏音量分别可调。
- 跟弹：用户键盘始终可以加入；自己的音符使用更亮的场景色，自动音符使用柔和白色，避免混淆。

### 曲库第一批

建议先精选 8–12 首明确可用的古典钢琴 MIDI，优先短曲和辨识度高的片段。第一版统一整理到 C2–B5 可视范围，确保自动按键与当前三八度键盘能一一对应。

每首曲目必须带：

- `title`、`composer`、`duration`
- `sourceUrl`、`license`、`attribution`
- `range`、`tempo`、`arranger`
- 经过人工听感确认的 MIDI/JSON 文件

[Mutopia Project](https://www.ibiblio.org/mutopia/index.html) 提供可下载的 MIDI 预览，并为每首作品单独标注许可，可作为第一批曲库候选来源。不能只因为作曲家年代久远就默认 MIDI 编配文件无版权。

## 3. 技术实现

### 文件解析

使用 [`@tonejs/midi`](https://www.npmjs.com/package/@tonejs/midi) 解析标准 MIDI 文件，构建统一的音符事件：

```ts
type SongNote = {
  midi: number;
  startSeconds: number;
  durationSeconds: number;
  velocity: number;
  hand?: "left" | "right";
};
```

内置曲目在构建阶段转成紧凑 JSON，线上播放时不重复解析；用户本地导入 `.mid` 可以在浏览器内解析，不上传服务器。

### 调度方式

继续使用当前 `AudioContext` 与钢琴采样，不把自动演奏改成普通 `setTimeout`：

1. 播放时记录 `audioContext.currentTime` 为曲目零点。
2. 每 25ms 查看未来 150ms 的音符事件。
3. 使用明确的 AudioContext 时间调用采样 `source.start(audioTime)`。
4. 视觉动画按同一个音频时间对齐，但仍在声音调度之后创建。
5. 自动声部使用 `auto:<note-id>`，用户声部继续使用键盘 code，两者互不抢占。

这种结构延续 Tone.js Transport 所采用的“把准确音频时间传给调度回调”的原则，但无需把当前稳定的低延迟音频链整体迁移到 Tone.js。[Tone.js 官方文档](https://tonejs.github.io/)也强调普通 JavaScript 回调并不精确，应使用传入的音频时间安排声音。

### 视觉反馈

- 自动演奏：柔和白金色光尘、琴键低亮度按下。
- 用户跟弹：当前场景强调色、琴键高亮度按下。
- 两者同音时叠加成更明亮的琴键边缘，不重复制造过量粒子。
- 光尘统一飞到相同最高处，继续遵守本版本规则。

## 4. 实施阶段

### V1：可听、可跟弹

- 8–12 首审核过授权的内置曲目。
- 播放、暂停、进度、速度、循环和双音量。
- 自动演奏与用户演奏并发。
- 曲目音符驱动琴键和场景光尘。

### V2：练习能力

- 左手、右手声部单独静音。
- A–B 片段循环已于 `v0.16.0` 完成。
- Web Audio 节拍器与 MIDI 速度/拍号同步已于 `v0.16.0` 完成。
- “等待我弹对再继续”的跟随练习模式。
- 本地导入 `.mid`，文件只留在当前浏览器会话。

### V3：乐谱与个人曲库

- 如确有乐谱阅读需求，再接入 [OpenSheetMusicDisplay](https://opensheetmusicdisplay.github.io/classdoc/classes/OpenSheetMusicDisplay.html) 渲染 MusicXML 与跟随光标。
- 本地文件夹曲库、搜索和动态播放队列已于 `v0.15.0` 完成；后续增加收藏、最近播放和可选的 IndexedDB 目录授权记录。

## 5. 验收标准

- 自动播放五分钟，音符相对音频时钟无可感知漂移。
- 自动和弦、用户和弦同时演奏时无明显断音或抢音。
- 用户按键延迟不因欣赏模式调度器而上升。
- 暂停后不残留自动声部，恢复位置准确。
- 切换场景和音色不导致曲目进度丢失。
- 每首内置曲目均有来源、许可证、编配者和人工听感确认记录。
- 最终使用有线耳机进行一次人工低延迟与音量平衡验收。

## 6. 产品决策

推荐先做 V1，不立即加入复杂乐谱。它能最短路径验证三个核心问题：用户是否愿意听、是否愿意跟弹、曲库是否能形成持续打开产品的理由。验证成立后，再投入声部分离、跟随练习和 MusicXML。
