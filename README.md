# 乐境 MeloVista · 沉浸式六音区钢琴

乐境（MeloVista）是一间支持中文和英文界面的沉浸式浏览器琴房。电脑键盘演奏覆盖 C1–B6，鼠标和触摸可直接弹奏页面上全部六个音区，并支持多点和滑奏。页面同时显示 C1 到 C6 六组连续的真实钢琴键盘：C4、C5 固定映射电脑键盘，低音组在 C2/C3 间切换，扩展组在 C1/C6 间切换；未启用组仍可用鼠标或触摸弹奏。可在沧海听风、山湖静语、雨夜伴灯与暮光之城四个动态场景间切换。

声音使用 Web Audio API 直接调度，音频先启动、视觉粒子后创建。采样加载并解码到内存后，演奏热路径不再请求网络。场景视频已离线合成为“正放 → 倒放”的静音乒乓循环，首尾回到同一画面后直接循环，不再使用可见的长交叉淡化；对应静态 poster 负责首屏和失败兜底，系统启用“减少动态效果”时只显示静态画面。

所有琴键产生的光尘都会抵达相同最高处，低音与高音只改变声音，不再改变粒子上升终点。场景与音色均使用自定义玻璃弹层，避免浏览器原生下拉框破坏沉浸感。

## 欣赏模式与曲库

顶部“曲库”打开欣赏模式。自动演奏通过 MIDI 音符事件复用现有 Web Audio 钢琴采样，支持播放/暂停、上一首/下一首、进度跳转，以及顺序播放、单曲循环、随机播放三种模式和 `0.75× / 1× / 1.25× / 1.5× / 2×` 速度。顺序播放会在曲库末尾回到第一首，随机播放不会连续抽到当前曲目。自动音符使用柔和白金色琴键与光尘，用户键盘保持场景强调色，两者可以同时演奏。

内置曲目：

- 圣诞快乐，劳伦斯先生 / Merry Christmas, Mr. Lawrence
- 蒲公英的约定 / Dandelion's Promise
- 花海 / Flower Sea
- 致爱丽丝 / Für Elise

《致爱丽丝》使用 Mutopia Project 明确标注为 Public Domain 的 MIDI；《圣诞快乐，劳伦斯先生》和《蒲公英的约定》使用项目所有者提供并明确允许随站点发布的 MIDI 文件。《花海》根据项目所有者提供的三页钢琴谱图片在本地识谱生成，并按小节人工修复 OMR 节奏错位、跨页漂移和第 41–48 小节的 `8va`；最终文件经过音域、速度、轨道、时长、重复事件与反向排谱校验。

曲库支持多选 MIDI、选择整个文件夹以及拖拽 `.mid` / `.midi` 文件。页面会逐首解析曲名、路径、时长、BPM 和音符数量；单个坏文件只会被跳过，不会让整批导入失败。本地曲目可搜索，并与四首内置曲目共同进入上一首/下一首、顺序播放和随机播放队列。所有文件只在当前浏览器会话读取和解析，不上传服务器、不写入项目，刷新页面后需要重新选择文件夹。

## 节拍器与练习

顶部“练习”打开节奏训练面板。节拍器支持 `40–208 BPM`、`2/4`、`3/4`、`4/4`、`6/4` 拍号、每小节第一拍重音和常用速度预设；“跟随当前曲目”会读取 MIDI 的首个速度，并按整首覆盖时长选择主拍号，避免把弱起小节误当成全曲拍号。节拍声使用 Web Audio 提前排程，不依赖普通 JavaScript 计时器临时出声。

播放曲目时可在任意进度设置 A 点和 B 点，启用后会在音频时钟到达 B 点时释放已排程音符并从 A 点重新开始，适合反复练习短句。切换曲目会自动清除旧循环，页面进入后台时会停止节拍器，避免意外持续出声。

## 环境声

四个动态背景的原始 AAC 音轨已离线提取为独立无缝循环环境音：素材从中点切开重排，并在原始首尾处做等功率交叉淡化，浏览器解码到 Web Audio 缓冲区后进行采样级循环。页面首次进入时保持静音；用户主动点击“启动音频”后，以 `0.28` 为基础音量开启当前场景环境声。雨夜源文件的响度远高于其他场景，已在素材层降低 `27.5dB` 至约 `-55 LUFS`，并使用独立版本化 URL 避免旧缓存。切换场景时，新旧环境音交叉淡化 `0.8` 秒；顶部“环境声”按钮可随时单独开启或关闭，不影响钢琴与 MIDI 播放。

## 启动

```powershell
npm.cmd install
npm.cmd run dev
```

打开终端显示的本地地址，选择音色后可点击右上角“加载并启动”，再使用电脑键盘演奏；也可以直接点击或触摸琴键，页面会在首次手势时自动准备音源。

## 多语言

顶部控制条可在中文与 English 之间即时切换，场景、音色、状态提示、性能信息、琴键辅助说明和无障碍文本会同步更新。选择保存在当前设备，并写入 `?lang=zh` 或 `?lang=en`，便于分享指定语言的界面。

## 沉浸模式

点击顶部的“沉浸演奏”，页面会屏蔽无关按键，仅保留 48 个琴键、`Space` 低音区切换、小键盘 `+` 扩展音区切换和左 `Alt` 长短音切换。退出沉浸模式使用页面按钮。支持 Keyboard Lock 的浏览器会同时尝试全屏键盘锁定；系统保留快捷键仍可能由操作系统接管。

## 音色

- 原声：Salamander Grand Piano 原声采样
- 明亮：同一钢琴采样叠加高频搁架滤波
- 小提琴：真实弓弦采样
- 钢丝弦吉他：真实拨弦采样
- 萨克斯：真实管乐采样

原声与明亮钢琴共用一套约 1.2MB 的采样；其余音色仅在第一次选择时按需加载，避免拖慢首次打开。

## 键位

- C4–B4 全音：`· 1 2 3 4 5 6`
- C4–B4 半音：`Tab Q E R T`
- C5–B5 全音：`7 8 9 0 - = Backspace`
- C5–B5 半音：`U I P [ ]`
- C3–B3 全音：`A S D F G H J`
- C3–B3 半音：`Z X V B N`
- C1–B1 全音：`Insert Home PageUp NumLock Numpad / * -`
- C1–B1 半音：`Delete End Numpad 7 8 9`
- `Space`：将低音组在 C2–B2 与 C3–B3 间切换，默认显示 C2，使左侧与 C1 组成连续的 C1–B2 可视范围
- 小键盘 `+`：将扩展组在 C1–B1 与 C6–B6 间切换；切换到 C6 时该组移动到键盘最右侧
- 左侧 `Alt`：在短音与长音之间切换；短音尾音约 2–3 秒，长音尾音最长约 9 秒；右侧 Alt 不响应

页面会显示最近一次键盘事件到音频调度的 JS 耗时、最近 120 次 P95，以及浏览器报告的基础音频延迟。JS 调度耗时不等于扬声器最终出声的端到端延迟，最终听感需要使用内置扬声器或有线耳机人工确认。
